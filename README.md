# 🧠 CNN Image Classifier with TensorFlow & Streamlit

A polished, end-to-end demo project for image classification. This project uses a custom Convolutional Neural Network (CNN) built with TensorFlow/Keras to classify images into one of the 10 CIFAR-10 categories. The entire workflow is neatly structured inside Jupyter Notebooks for an interactive, educational experience.

---

## 📂 Project Structure

This repository uses a **Jupyter Notebook First** approach. The Python scripts (`.py`) are automatically generated by the notebooks.

```text
tensorflow_cnn_streamlit_demo/
├── prediction_utils.ipynb   # 1. Generates prediction functions
├── train_model.ipynb        # 2. Builds and trains the CNN model
├── app.ipynb                # 3. Generates and runs the Web UI
├── requirements.txt         # Pinned project dependencies
├── README.md                # Project documentation
├── models/                  # Saved CNN models (.keras)
└── assets/                  # Training charts and UI diagrams
```
> **Note:** Files like `app.py` and `utils/prediction.py` are ignored by Git because they are dynamically created when you run the notebooks.

## 🚀 How to Run the Project (Step-by-Step)

Follow these exact steps to clone the project, generate the necessary files, and start the web interface.

### Step 0: Clone and Install Dependencies
Open your terminal and run:
```bash
git clone https://github.com/ElysiaTheElysier/Simple_CNN.git
cd Simple_CNN
pip install -r requirements.txt
```

### Step 1: Generate Utilities
Open the **`prediction_utils.ipynb`** notebook in VS Code or Jupyter.
- Click **Run All Cells**.
- *This creates the `utils/prediction.py` file required for the Streamlit app.*

### Step 2: Train the Model (Optional but Recommended)
Open the **`train_model.ipynb`** notebook.
- Click **Run All Cells**.
- *This downloads the CIFAR-10 dataset, trains the CNN (up to 30 epochs with Early Stopping), saves the best weights to `models/cnn_cifar10_model.keras`, and updates the charts in the `assets/` folder.*
- *If you skip this step, the app will use the pre-trained model included in this repository.*

### Step 3: Launch the Web UI
Open the **`app.ipynb`** notebook.
- Click **Run All Cells**.
- *This generates the `app.py` script and automatically executes `!streamlit run app.py`.*
- The Streamlit web interface will open in your browser at `http://localhost:8501`.

*(Alternatively, after running Step 1 and Step 3's notebook once, you can just type `streamlit run app.py` directly in your terminal).*

## 🛠️ Troubleshooting

- **VS Code Notebook Hangs / Keeps Loading:** If the Jupyter kernel in VS Code gets stuck on the first run, click the **Restart Kernel** button at the top of the notebook, hit **Clear All Outputs**, and try again.
- **Missing Module Error:** If Streamlit crashes with `ModuleNotFoundError: No module named 'utils.prediction'`, it means you skipped Step 1. Go back and run `prediction_utils.ipynb` first.
- **Old Code Running / Deprecation Warnings:** If you pull new updates from Git but the app behaves strangely, make sure you **Run All Cells** in the notebooks again so they can overwrite the old `.py` files with the newest code.

## 🤖 Model Details

- **Dataset:** CIFAR-10 (60,000 images, 32x32 RGB, 10 classes).
- **Architecture:** 
  - **Data Augmentation:** Random Flip (Horizontal), Random Rotation (0.05) to improve generalization.
  - **Feature Extraction:** 3x `Conv2D` layers (32, 64, 128 filters) paired with `MaxPooling2D`.
  - **Classification:** `Flatten` -> `Dense(128)` -> `Dropout(0.4)` -> `Dense(10, Softmax)`.
- **Training Strategy:** Adam optimizer, Sparse Categorical Crossentropy loss. Trains for a maximum of 30 epochs, utilizing **Early Stopping** (patience=4) to prevent overfitting and capture the best weights.
